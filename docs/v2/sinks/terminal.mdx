Prints each event to the terminal with color-coded, human-readable formatting (the same look
as `oura watch`). Use it when you want to *watch* a pipeline live rather than feed events into
another program.

## Configuration

```toml title="daemon.toml"
[sink]
type = "Terminal"
throttle_min_span_millis = 500
wrap = true
```

- `type` (required): the literal value `Terminal`.
- `throttle_min_span_millis` (optional, default = `500`): minimum delay, in milliseconds,
  between printed events. Throttling keeps the output readable instead of dumping a whole
  block's events at once.
- `wrap` (optional, default = `false`): when `true`, long lines wrap; otherwise they're
  truncated to the terminal width.
- `adahandle_policy` (optional): the minting policy id (hex) of [ADA Handle](https://adahandle.com/).
  When set, the sink resolves and displays handles found in outputs for a friendlier rendering.
  Omit it to disable handle resolution.
